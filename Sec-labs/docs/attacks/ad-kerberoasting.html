Identity Attack: Kerberoasting

MITRE T1558.003: Exploiting legacy Active Directory Service Principal Names (SPNs).

‚öîÔ∏è The Attack (Red Team)

Objective: Request a TGS (Ticket Granting Service) ticket for a service account. The ticket is encrypted with the service account's NTLM hash. If we can crack the ticket offline, we get the password.

# 1. Enumerate SPNs (Find targets) GetUserSPNs.py domain.local/user:password -outputfile hashes.txt # 2. Crack the Hash (Offline) hashcat -m 13100 hashes.txt /usr/share/wordlists/rockyou.txt

üõ°Ô∏è The Defense (Zero Trust)

Why does this work? Because Active Directory trusts any authenticated user to request a ticket for any service. Zero Trust demands we Verify Explicitly.

Detection (SIEM)

Monitor Event ID 4769 (Kerberos Service Ticket Operation).

  • Alert: Ticket Encryption Type = 0x17 (RC4-HMAC) - Modern systems should use AES.
  • Alert: Excessive TGS requests from a single user in a short time.

Mitigation

1. MSA (Managed Service Accounts): Use gMSA with 120-character random passwords.

2. AES Encryption: Force AES-256 for Kerberos to make offline cracking exponentially harder.
